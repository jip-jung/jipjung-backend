## 📋 문서 개요

이 문서는 집-중(JipJung) 프로젝트의 전체 기능에 대한 체계적인 테스트 계획을 정의합니다.
사용자 플로우 기반으로 테스트 시나리오를 구성하여 누락 없이 전체 기능을 검증합니다.

### 테스트 범위
| 영역 | 상태 | 설명 |
|------|------|------|
| 인증 (Authentication) | ✅ 수동 테스트 완료 | 회원가입, 로그인, 로그아웃 |
| 온보딩 (Onboarding) | ✅ 수동 테스트 완료 | 프로필 설정, 드림홈 설정 |
| 저축 (Savings) | ⏳ 테스트 필요 | 입금/출금, 목표 달성 |
| 게이미피케이션 | ⏳ 테스트 필요 | 경험치, 레벨, 스트릭 |
| 대시보드 (Dashboard) | ⏳ 테스트 필요 | 통합 데이터, 집 성장 |
| DSR 시뮬레이션 | ⏳ 테스트 필요 | LITE/PRO 모드 |
| AI 관리실 | ⏳ 테스트 필요 | AI 분석, 판결 |
| 컬렉션 | ⏳ 테스트 필요 | 완료 기록, 여정 리플레이 |

## 🔐 Flow 1: 인증 플로우 ✅ 수동 테스트 완료

### 1.1 회원가입

| ID | 테스트 케이스 | 사전조건 | 테스트 단계 | 예상 결과 | 상태 |
|----|--------------|---------|------------|----------|------|
| AUTH-001 | 유효한 정보로 회원가입 | 미가입 이메일 | 1. /register 접속<br>2. 이메일/비밀번호/닉네임 입력<br>3. 가입 버튼 클릭 | 201 Created, 로그인 페이지로 이동 | ✅ |
| AUTH-002 | 중복 이메일 회원가입 | 이미 가입된 이메일 | 1. 기존 이메일로 가입 시도 | 409 Conflict 에러 | ✅ |
| AUTH-003 | 비밀번호 형식 검증 | - | 1. 8자 미만 비밀번호 입력 | 프론트 유효성 검사 실패 | ✅ |
| AUTH-004 | 필수 필드 누락 | - | 1. 이메일 없이 가입 시도 | 400 Bad Request | ✅ |

**관련 코드:**
- Backend: `AuthController.signup()`, `AuthService.signup()`
- Frontend: `RegisterView.vue`, `authStore.register()`

